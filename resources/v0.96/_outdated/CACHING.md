# Кэширование результатов PubMed API

## Обзор

Реализован механизм дискового кэширования результатов поиска PubMed для:
- **Ускорения повторных запросов** - результаты возвращаются мгновенно из кэша
- **Снижения нагрузки на PubMed API** - меньше запросов к внешнему API
- **Экономии времени** - особенно полезно при повторном анализе протоколов

## Компоненты

### 1. CacheManager (`modules/cache_manager.py`)

Базовый класс для управления кэшем с поддержкой:
- **TTL (Time To Live)** - автоматическая инвалидация старых записей
- **Thread-safe** - безопасное использование в параллельной обработке
- **Атомарные операции** - защита от повреждения данных
- **Статистика** - мониторинг использования кэша

### 2. PubMedCacheManager

Специализированный класс для кэширования PubMed результатов:
- Упрощенный API для работы с PubMed запросами
- Автоматическая генерация ключей на основе параметров поиска

### 3. Интеграция в PubMedClient

Кэширование автоматически интегрировано в `PubMedClient`:
- Проверка кэша перед выполнением запроса
- Сохранение результатов после получения
- Автоматическая очистка истекших записей при инициализации

## Как это работает

### Процесс кэширования

1. **При запросе к PubMed:**
   ```
   search_evidence(inn="metformin", indication="diabetes", dosage="500mg", route="oral")
   ```

2. **Генерация ключа кэша:**
   - Создается hash на основе всех параметров: `SHA256(inn + indication + dosage + route)`
   - Ключ используется как имя файла: `{hash}.json`

3. **Проверка кэша:**
   - Если файл существует и не истек TTL → возвращается из кэша
   - Если файл не существует или истек → выполняется запрос к API

4. **Сохранение результата:**
   - После получения результата от API → сохраняется в кэш
   - Файл содержит данные + метаданные (timestamp, TTL)

### Структура файла кэша

```json
{
  "data": {
    "rct_count": 5,
    "meta_analysis_count": 2,
    "articles": [...],
    "total_found": 10,
    "from_cache": false
  },
  "timestamp": "2024-01-15T10:30:00",
  "ttl_days": 30,
  "cache_key": "abc123..."
}
```

## Настройки

### В config.py

```python
enable_cache: bool = True           # Включить/выключить кэширование
cache_ttl_days: int = 30            # Время жизни кэша в днях
```

### Переменные окружения

- `ENABLE_CACHE` - включить/выключить кэширование (true/false)
- `CACHE_TTL_DAYS` - время жизни кэша в днях (по умолчанию 30)

### Расположение кэша

Кэш хранится в: `data/cache/pubmed/`

Каждый файл имеет формат: `{hash}.json`

## Использование

### Автоматическое использование

Кэширование работает автоматически при использовании `PubMedClient`:

```python
from modules.api_clients import PubMedClient

client = PubMedClient()

# Первый запрос - идет к API
result1 = client.search_evidence("metformin", "diabetes")

# Второй запрос с теми же параметрами - из кэша (мгновенно)
result2 = client.search_evidence("metformin", "diabetes")
```

### Ручное управление кэшем

```python
from modules.cache_manager import PubMedCacheManager
from pathlib import Path

cache = PubMedCacheManager(
    cache_dir=Path("data/cache/pubmed"),
    ttl_days=30,
    enabled=True
)

# Получение из кэша
cached = cache.get_pubmed_result("metformin", "diabetes", "500mg", "oral")

# Сохранение в кэш
cache.set_pubmed_result("metformin", "diabetes", result, "500mg", "oral")

# Очистка истекших записей
expired_count = cache.clear_expired()

# Полная очистка кэша
deleted_count = cache.clear_all()

# Статистика
stats = cache.get_stats()
print(f"Всего записей: {stats['total_files']}")
print(f"Действительных: {stats['valid_files']}")
print(f"Размер кэша: {stats['total_size_mb']} MB")
```

## Очистка кэша

### Автоматическая очистка

- При инициализации `PubMedClient` автоматически очищаются истекшие записи
- Старые записи (старше TTL) удаляются автоматически

### Ручная очистка

1. **Очистка истекших записей:**
   ```python
   cache.clear_expired()
   ```

2. **Полная очистка кэша:**
   ```python
   cache.clear_all()
   ```

3. **Удаление директории:**
   ```bash
   rm -rf data/cache/pubmed/
   ```

## Отключение кэширования

Если нужно отключить кэширование:

1. **Через переменную окружения:**
   ```bash
   set ENABLE_CACHE=false
   ```

2. **Или в коде:**
   ```python
   config.enable_cache = False
   ```

## Рекомендации

1. **Используйте кэширование** - значительно ускоряет повторные запросы
2. **Настройте TTL** - 30 дней оптимально для большинства случаев
3. **Мониторьте размер кэша** - периодически очищайте старые записи
4. **Не отключайте кэш** - это замедлит работу при повторных анализах

## Производительность

### Без кэша:
- Запрос к PubMed API: ~2-5 секунд
- При 10 препаратах: ~20-50 секунд

### С кэшем:
- Запрос из кэша: ~0.001 секунды (мгновенно)
- При 10 препаратах (все в кэше): ~0.01 секунды

**Ускорение: в 2000-5000 раз для повторных запросов!**
