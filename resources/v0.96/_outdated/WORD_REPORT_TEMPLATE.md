# Генерация Word отчета по шаблону

## Обзор

Реализована поддержка генерации Word отчета по шаблону `example.docx` из папки `data/output_example/`.

## Шаблон

**Расположение:** `data/output_example/example.docx`

**Структура шаблона:**
- Заголовок: "КЛИНИЧЕСКИЙ ПРОТОКОЛ НАЗВАНИЕ" (центрированный, жирный)
- Таблица: 5 колонок, стиль "Table Grid"
- Заголовки таблицы: все колонки с одинаковым текстом
- Выравнивание: JUSTIFY для всех ячеек

## Автоматическое использование

При генерации Word отчета система автоматически:
1. Проверяет наличие шаблона `data/output_example/example.docx`
2. Если шаблон найден - использует его как основу
3. Если шаблон не найден - создает отчет в стандартном формате

## Формат отчета по шаблону

### Заголовок
- Текст: "АНАЛИЗ КЛИНИЧЕСКОГО ПРОТОКОЛА ПО ПРЕПАРАТАМ"
- Форматирование: центрированный, жирный (как в шаблоне)

### Таблица (5 колонок)

**Заголовки:**
- Все 5 колонок: "Сведения о препаратах клинического протокола"
- Выравнивание: JUSTIFY
- Шрифт: обычный (не жирный, как в шаблоне)

**Первая строка данных:**
- Общая информация: "В результате проведенной методологической оценки клинического протокола проанализировано N препаратов."

**Строки данных по препаратам:**
- **Колонка 1:** Наименование препарата
- **Колонка 2:** МНН (англ и рус, если есть)
- **Колонка 3:** УД (системный GRADE)
- **Колонка 4:** Статус регистрации и одобрения (BNF, WHO EML, FDA, EMA, Казахстан)
- **Колонка 5:** Дополнительная информация (дозировка, способ введения, количество исследований)

**Выравнивание:** Все ячейки - JUSTIFY (как в шаблоне)

### Детальная информация

После таблицы добавляется раздел "ДЕТАЛЬНАЯ ИНФОРМАЦИЯ ПО ПРЕПАРАТАМ" с подробными данными по каждому препарату.

## Использование

### Автоматическое (рекомендуется)

Шаблон используется автоматически при запуске анализа:

```bash
python main.py -i "data/inputs/protocol.docx" -o "data/outputs/report.xlsx" --indication "Заболевание"
```

Система автоматически:
- Проверяет наличие шаблона
- Использует его если найден
- Создает отчет в формате шаблона

### Ручное указание шаблона

```python
from modules.report_generator import generate_word_report
from pathlib import Path

template_path = Path("data/output_example/example.docx")
generate_word_report(
    all_verified_data=data,
    output_path=Path("output.docx"),
    protocol_summary=summary,
    template_path=template_path
)
```

## Формат без шаблона

Если шаблон не найден, используется стандартный формат:
- Таблица: 4 колонки
- Стиль: "Light Grid Accent 1"
- Заголовки: жирные, центрированные
- Выравнивание: CENTER для заголовков

## Настройка

### Изменение пути к шаблону

В `main.py` можно изменить путь к шаблону:

```python
template_path = Path("data/output_example/example.docx")
# или
template_path = Path("custom/path/to/template.docx")
```

### Отключение использования шаблона

Чтобы использовать стандартный формат без шаблона:

```python
generate_word_report(
    all_verified_data=data,
    output_path=Path("output.docx"),
    template_path=None  # Не использовать шаблон
)
```

## Структура данных в таблице

### Колонка 1: Наименование препарата
- Источник: `original_name` из данных верификации
- Пример: "Метформин 500 мг"

### Колонка 2: МНН
- Формат: `{inn_english}\n({inn_russian})` если есть русский
- Или: `{inn_english}` если только английский
- Пример: "metformin\n(метформин)"

### Колонка 3: УД (системный GRADE)
- Формат: `УД: {system_ud}`
- Пример: "УД: A"

### Колонка 4: Статус регистрации
- Формат: список через запятую
- Пример: "BNF, WHO EML, FDA, Казахстан"

### Колонка 5: Дополнительная информация
- Формат: многострочный текст
- Содержит:
  - Дозировка (если указана)
  - Способ введения (если указан)
  - Количество исследований (мета-анализы, РКИ)
- Пример:
  ```
  Дозировка: 500 мг
  Способ: перорально
  Мета-анализы: 2; РКИ: 5
  ```

## Рекомендации

1. **Используйте шаблон** - обеспечивает единообразный формат отчетов
2. **Проверьте шаблон** - убедитесь что `example.docx` существует
3. **Настройте шаблон** - можно изменить форматирование в шаблоне
4. **Резервная копия** - сохраните оригинальный шаблон

## Технические детали

### Обработка шаблона

1. Загрузка шаблона через `Document(str(template_path))`
2. Сохранение настроек документа (поля, стили)
3. Очистка содержимого (параграфы и таблицы)
4. Обновление заголовка
5. Создание новой таблицы в стиле шаблона
6. Заполнение данными

### Сохранение форматирования

- Поля документа сохраняются из шаблона
- Стили таблицы соответствуют шаблону
- Выравнивание текста как в шаблоне (JUSTIFY)
