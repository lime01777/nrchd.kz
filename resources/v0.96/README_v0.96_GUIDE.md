# Руководство пользователя для Версии 0.96

Это подробная инструкция по установке, настройке и использованию анализатора клинических протоколов версии **v0.96**.

---

## 🚀 1. Установка

### Требования
- Операционная система: Windows (рекомендуется), macOS или Linux.
- **Python 3.10** или выше. [Скачать Python](https://www.python.org/downloads/)

### Шаги установки

1.  **Откройте папку проекта**:
    Перейдите в папку `v0.96` (например, `Documents/v0.96`).

2.  **Запустите автоматическую установку**:
    Найдите файл `setup_install.bat` и запустите его двойным кликом.
    
    *Что делает этот скрипт:*
    - Проверяет наличие Python.
    - Создает виртуальное окружение (`venv`).
    - Активирует его.
    - Устанавливает все необходимые библиотеки из `requirements.txt`.

    > **Важно:** Если окно закроется с ошибкой или скрипт не сработает, убедитесь, что Python добавлен в PATH при установке.

---

## ⚙️ 2. Настройка (ОБЯЗАТЕЛЬНО)

Перед запуском необходимо настроить API ключи.

1.  **Создайте файл `.env`**:
    В папке `v0.96` создайте новый текстовый файл и назовите его `.env` (без расширения .txt).
    
    > *Совет: Если у вас уже есть файл `.env` от предыдущих версий, скопируйте его в эту папку.*

2.  **Заполните `.env`**:
    Откройте файл в Блокноте и добавьте следующие строки (замените значения на ваши реальные ключи):

    ```env
    # Google Gemini API (для анализа текста)
    GEMINI_API_KEY="ваш_ключ_gemini_от_google_ai_studio"
    
    # PubMed API (для поиска исследований)
    PUBMED_API_KEY="ваш_ключ_pubmed"
    PUBMED_EMAIL="ваш_email@example.com"
    PUBMED_TOOL="ProtocolAnalyzer"
    
    # Настройки
    GEMINI_MODEL="gemini-3-flash-preview"
    ```

    > **Как получить ключи:**
    > - **Gemini API Key**: [Google AI Studio](https://aistudio.google.com/app/apikey)
    > - **PubMed API Key**: [NCBI Account Settings](https://www.ncbi.nlm.nih.gov/account/settings/)

## 🖥️ 3. Использование

### Способ А: Веб-интерфейс (Рекомендуется)

Самый простой способ работы.

1.  Запустите файл `run_web_ui.bat`.
2.  Автоматически откроется браузер с адресом `http://localhost:8501`.
3.  **В интерфейсе**:
    - Перетащите файл протокола (Word .docx) в область загрузки.
    - Введите название показания (заболевания, например "Сахарный диабет").
    - Нажмите кнопку **"Начать анализ"**.
4.  Дождитесь завершения. Отчет будет доступен для скачивания.

### Способ Б: Через командную строку (Для продвинутых)

Если вы хотите проанализировать файл вручную или пакетно.

1.  Положите ваши `.docx` файлы в папку `data/inputs`.
2.  Откройте командную строку (cmd) в папке проекта.
3.  Активируйте окружение: `venv\Scripts\activate`
4.  Запустите анализ:
    ```bash
    python main.py -i "data/inputs/ваш_файл.docx" -o "data/outputs/отчет.xlsx" --indication "Название болезни"
    ```

## 🔧 Исправление Ошибок

### Ошибка 429 (Resource Exhausted)
Если вы видите ошибку `429` от Gemini, это значит, что превышен лимит запросов (часто бывает на бесплатном тарифе).

**Решение в v0.96:**
Мы уже внедрили автоматическую защиту. Система теперь:
1.  Ждет 10 секунд перед повторной попыткой.
2.  При повторной ошибке увеличивает время ожидания (10с -> 20с -> 40с...).
3.  Делает до 5 попыток.

Если ошибки продолжаются, попробуйте сменить API ключ или подождать пару минут.

Все отчеты сохраняются в папку `data/outputs`:
