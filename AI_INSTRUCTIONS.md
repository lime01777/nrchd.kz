# План настройки и запуска AI-окружения

Для того чтобы анализ протоколов работал локально и виртуальное окружение всегда было активно, выполнены следующие действия:

## 1. Подготовка виртуального окружения
- В папке `resources/v0.96` создано/обновлено окружение `.venv_linux`.
- Установлены все необходимые зависимости из `requirements.txt` (pandas, streamlit, google-generativeai и др.).
- Проверены API ключи в файле `.env`.

## 2. Автоматизация запуска (Queue Worker)
Для работы через админку Laravel (`/admin/ai-protocols`) необходимо, чтобы в фоне работал обработчик очереди:
- Был запущен процесс: `php artisan queue:work`.
- Именно этот процесс использует виртуальное окружение для вызова Python-скриптов.

## 3. Вспомогательный скрипт
Создан файл `start-ai.sh` в корне проекта.
- **Назначение:** Одной командой проверяет окружение и запускает очередь.
- **Как запустить:** Откройте терминал и введите `./start-ai.sh`.

## 4. Как держать окружение "всегда запущенным"
Чтобы сервис был готов к работе в любое время:
1. Запустите `./start-ai.sh`.
2. Оставьте это окно терминала открытым.
3. Теперь при нажатии кнопки "Анализировать" в админке, задача будет подхватываться мгновенно.
